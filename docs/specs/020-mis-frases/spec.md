## Why

Mis frases is the third editorial pillar of the product, alongside MCQs (015) and Phrase Autopsy (016). 019 made saves durable — the backend now persists chunks with Claude-generated speaking prompts and exposes `GET /api/chunks` — but no UI consumes that data. Today the `/chunks` route is a one-line placeholder (`webapp/src/routes/chunks.tsx`), the homepage KPI "Frases guardadas" is hard-coded to `0` (`routes/index.tsx:148-150`), and the save toggle in `AutopsyPanel` is a session-local `Set<string>` that forgets every refresh (`routes/listen.$id.tsx:92`).

020 ships the screen Ana actually opens to *use* her saved phrases — rotating speaking prompts she practices out loud — and connects the three places the chunk library touches the rest of the app: the Inicio KPI, the AutopsyPanel save toggle, and the Mis frases route itself. After 020, every "guardar" in the app writes a real row, that row is visible across reloads, and Ana can return to it later to practice.

### Consumer Impact

- **Ana**, the single B2 learner. Today the right rail's save toggle is theatre — it flips colour and forgets on reload; the KPI lies (`0`); the Mis frases route is a placeholder. After 020: tap "guardar" on an autopsy → the phrase lands in her library with 2-4 practice prompts (already generated by 019 at save time), is browsable on the Mis frases screen, contributes to the homepage count, and jumps back to its source video on click. The recording UX is visually present but doesn't capture audio yet (021).
- **019 (backend chunks).** First real consumer — until 020, 019's endpoints have no caller in the app, only in tests. 020 exercises the cache-hit semantics (same phrase saved twice → one row), the response shape (`prompts`, `source_title`, `start_time`, `created_at`), and confirms the contract.
- **016 (Phrase Autopsy).** The save toggle stops being decorative. The visual contract doesn't move (`Guardar en biblioteca` ↔ `Guardada en biblioteca`); only the handler swaps from `setSavedPhrases((s) => …)` to a real mutation against `POST /api/chunks` with cache-invalidation on the chunks query.
- **014 (Inicio).** The "Frases guardadas" KPI becomes the real count via one additional `useQuery` against `GET /api/chunks`.
- **021 (recordings) inherits the visual contract.** The recording button ships as a stub (toggle state + auto-stop timer, no audio), tagged `temporal · ver 021` the same way 016 tagged its trigger card `temporal · ver 018`. When 021 lands, only the handler changes — the button, the state, and the layout stay.

### Roadmap Fit

- **Depends on:** 019 (chunks endpoints — `POST` / `GET` / `DELETE`), 015 (listen route — needs a small `?t=` deep-link addition), 016 (AutopsyPanel — its save toggle is rewired).
- **Soft-depends on:** nothing. 017 / 018 are unaffected.
- **Unblocks:** 021 (recordings — the recording UI shape is established here; 021 only adds capture + upload + playback). The homepage KPI stops being a placeholder, which is a small step toward 022 (user-profile + real weekly KPIs).
- **Doesn't block:** any backend feature.

### Non-Goals

- **Real audio recording / capture / upload / playback.** 021 ships that. The button is a visual stub.
- **Real mastery signal.** No mastery column in 019; no signal until 021. Dots are static 0/5.
- **"Necesitan práctica" filter actually filtering.** Inert until 021.
- **"Empezar sesión de práctica" routing or session logic.** No spec defines it. Stub button only.
- **In-screen delete affordance on chunk cards.** Unsave is via the AutopsyPanel toggle on `/listen/{id}`, mirroring where the save originated. If Ana asks for an inline × later, it's one button + the `deleteChunk()` client method already in place.
- **Search / sort beyond newest-first.** Defer. The filter chips are the only top-level controls.
- **Per-video chunk listing on Escuchando.** 019 §Non-Goals — Mis frases is global.
- **Edit / regenerate prompts.** 019's cache is write-once; bad prompts → delete + re-save.
- **Cross-session prompt-rotation memory.** `promptIdx` is component-local. Navigating away resets to the first prompt. Persisting it would need either backend support or local storage; not worth a column in 020.
- **English glosses.** Permanent product position per OVERVIEW §Non-Goals. The design's `gloss: "around nine-ish"` field is dropped — 019's `ChunkResponse` already doesn't carry it.
- **Frontend automated tests.** Same deferral as 013–018. Manual smoke is the bar.
- **`POST /api/chunks` from Mis frases itself.** Saves only originate from AutopsyPanel. Mis frases is read-mostly.

### Open Questions

1. **`Necesitan práctica` chip UX when inert.** Three reasonable shapes:
   - **(a) Visually disabled** (lower opacity, `cursor: not-allowed`, tooltip `próximamente`). Clean, but the chip is still there visually using row space without paying its way.
   - **(b) Hidden until 021.** Cleanest screen; loses the design's three-filter rhythm.
   - **(c) Active but always shows "Nada por ahora" empty state below**. Honest but cute, and the empty state copy lies a bit ("nothing here" vs "feature not ready").
   - **Recommendation: (a).** Matches 016's `temporal · ver NNN` precedent of *showing* the deferred surface so the eventual hook-up is a small handler swap. The visual disabled-state telegraphs that the feature exists, just not yet.

2. **Recording-stub auto-stop timer — 3.5s like the design, or instant toggle?**
   - Design uses `setTimeout(..., 3500)` to simulate a short recording. Keeping the timer makes the stub feel real during manual smoke. Dropping it would make the button visibly stub-y (tap → tap-back), which is more honest but more annoying.
   - **Recommendation: keep 3.5s.** Matches design intent. When 021 lands, the timer is replaced by the real MediaRecorder lifecycle and the stub becomes the actual flow.

3. **Relative time on the chunk source — `hace 2 días` vs `12 de mayo`?**
   - Design uses relative ("2 days ago", "Today", "Yesterday", "1 week ago" — `data.js:204,217,229,...`).
   - Relative is warmer but rolls the cost of an i18n-y formatter (or hand-rolled Spanish: `hoy`, `ayer`, `hace N días`, `hace N semanas`).
   - **Recommendation: relative, hand-rolled.** A 15-line helper in `webapp/src/lib/formatting.ts`. Reused later by Inicio if 022 wants a "última escucha" line.

4. **Optimistic save vs pending-only.**
   - Save round-trip is bounded by Claude latency (1-3s typical from 019's spike). Optimistic feels great if it works, regresses badly if Claude fails.
   - **Recommendation: pending-only.** Disable the button + dim text while in-flight. On success, animate to the saved state. On error, revert + inline message. Matches the actual contract (the chunk isn't really saved until the response lands) and avoids needing a rollback animation. Re-evaluate after 020 is in use.

5. **Cache key for the AutopsyPanel "is this saved?" lookup.**
   - Need a stable predicate that matches 019's backend `normalize_phrase` (whitespace + casefold). If frontend and backend disagree, the save button could look unsaved after a save lands.
   - **Recommendation:** mirror the helper in TS — `webapp/src/lib/normalize-phrase.ts` exporting `normalizePhrase(s) = s.trim().replace(/\s+/g, " ").toLocaleLowerCase("es")`. Document that this MUST stay in sync with `src/repositories/autopsy_repository.py:normalize_phrase`. (The drift risk is low — both are 1-liners — but worth a comment.)

export function listChunks(): Promise<Array<Chunk>> {
  return api<Array<Chunk>>("/api/chunks");
}

export function saveChunk(body: ChunkSaveRequest): Promise<Chunk> {
  return api<Chunk>("/api/chunks", {
    method: "POST",
    body: JSON.stringify(body),

  return api<void>(`/api/chunks/${id}`, { method: "DELETE" });
}

#### 4. Route — `routes/chunks.tsx`

```tsx
export const Route = createFileRoute("/chunks")({ component: MisFrases });

function MisFrases() {
  const chunksQuery = useQuery({ queryKey: ["chunks"], queryFn: listChunks });
  const [filter, setFilter] = useState<"all" | "low" | "recent">("all");

      <p className="page-sub">Cada frase venía de algo que escuchaste. …</p>
      <ChunksFilterRow
        filter={filter}
        onFilter={setFilter}
        totalCount={chunksQuery.data.length}
      />

        {filtered.map((c) => <ChunkCard key={c.id} chunk={c} />)}
      </div>
    </div>

```

`applyFilter` is a pure helper next to the component:

```ts
function applyFilter(chunks: Array<Chunk>, filter: "all" | "low" | "recent") {
  if (filter === "all") return chunks;
  if (filter === "low") return chunks;  // inert until 021 — see Open Question #1
  // "recent": within the last 7 days
  const cutoff = Date.now() - 7 * 24 * 60 * 60 * 1000;
  return chunks.filter((c) => Date.parse(c.created_at) >= cutoff);
}
```

#### 8. Wiring `AutopsyPanel` to real persistence

In `routes/listen.$id.tsx`:

```ts
const chunksQuery = useQuery({ queryKey: ["chunks"], queryFn: listChunks });
const queryClient = useQueryClient();

const saveChunkMutation = useMutation({
  mutationFn: (body: ChunkSaveRequest) => saveChunk(body),
  onSuccess: () => queryClient.invalidateQueries({ queryKey: ["chunks"] }),
});

const deleteChunkMutation = useMutation({
  mutationFn: (id: number) => deleteChunk(id),
  onSuccess: () => queryClient.invalidateQueries({ queryKey: ["chunks"] }),
});

const savedChunkFor = (phrase: string) => {
  const key = normalizePhrase(phrase);
  return chunksQuery.data?.find(
    (c) => c.video_id === youtubeId && normalizePhrase(c.phrase) === key,
  );
};

const isSaved = autopsy?.state === "loaded"
  ? !!savedChunkFor(autopsy.entry.phrase)
  : false;

const onToggleSavedPhrase = (phrase: string, startTime: number) => {
  const existing = savedChunkFor(phrase);
  if (existing) {
    deleteChunkMutation.mutate(existing.id);
  } else {
    saveChunkMutation.mutate({ video_id: youtubeId, phrase, start_time: startTime });
  }
};
```

#### 12. Relative-time helper

`webapp/src/lib/relative-time.ts`:

```ts
export function formatRelative(iso: string): string {
  const ms = Date.now() - Date.parse(iso);
  const days = Math.floor(ms / (24 * 60 * 60 * 1000));
  if (days < 1) return "hoy";
  if (days === 1) return "ayer";
  if (days < 7) return `hace ${days} días`;
  if (days < 14) return "hace 1 semana";
  if (days < 30) return `hace ${Math.floor(days / 7)} semanas`;
  if (days < 60) return "hace 1 mes";
  return `hace ${Math.floor(days / 30)} meses`;
}
```

Hand-rolled, Spanish-only, ~15 lines. No `Intl.RelativeTimeFormat` dep — keeps the bundle lean and the copy fully under our control.

### Confidence

**Level:** Medium leaning High.

**Rationale:**

Every pattern this spec uses is already in the codebase: react-query (`useQuery` + `useMutation` + `invalidateQueries`) is exercised in 014 / 015; route-file components with skeleton + empty + error states are exercised in 014; aside-panel rewiring is exactly the shape 016 ships today (only the handler swaps). The chunks API is real (019) and the response shape is known.

Two specific items keep this from being High:

1. **`?t=` deep-link race.** The YouTube player's `isReady` event needs to fire *after* the search-param is parsed but *before* the user's first interaction. 015's `useYouTubePlayer` hook already exposes a ready state, so the one-shot effect against `player.isReady` should work, but it deserves a 15-min spike to confirm (a) `validateSearch` runs SSR-safely, (b) the effect doesn't re-seek on Hot Module Reload, (c) playback resumes cleanly without user-gesture blocking from browser autoplay policies.

2. **Empty-body 204 from `DELETE /api/chunks/{id}`.** The existing `api()` wrapper always calls `await response.json()` (`webapp/src/lib/api.ts:22`). On a 204 with no body that throws. One-line fix (check `response.status === 204` before parsing, return `undefined as T`), but a real change in shared client code that ripples.

If both spike cleanly, confidence rises to High.

**Validate before proceeding:**

- 15-min spike: add `validateSearch` to `/listen/$id`, navigate from `/chunks` with `?t=42`, confirm the player seeks + plays on first ready without re-seeking on re-render.
- Confirm the `api()` empty-body fix doesn't regress any existing caller (`saveProgress` returns JSON, `explainPhrase` returns JSON, `getVideo` returns JSON — only the new `deleteChunk` returns nothing).

### Key Decisions

- **Maximalist visual port.** Every design affordance ships, with deferred-feature ones (mastery, recording, "Necesitan práctica", "Empezar sesión") as honest visual stubs marked `temporal · ver NNN` or `Disponible con 021`. Reasoning: 016's precedent established that a visual contract shipped now is cheaper to evolve than one withheld; the stubs lie *less* than zero UI does (the screen exists, the saved phrases render).
- **No in-screen delete on chunks.** Unsave originates from AutopsyPanel — same place save originated. Symmetric with 016's mental model. `deleteChunk()` exists in the API client but no Mis frases card surfaces a × button. Reconsider when Ana asks.
- **Single `["chunks"]` query key across Inicio + Mis frases + AutopsyPanel.** React-query dedupes; one mutation invalidates all three views. Same pattern 015 uses for `["video-progress", youtubeId]`.
- **`isSaved` derived from query data, not local state.** Removes 016's session-local `Set<string>` and the entire "saved state resets on reload" footnote.
- **Pending-only save, not optimistic.** Bounded-but-perceptible Claude latency (1-3s) — optimistic flip would feel great but rolling it back on a 502 is uglier than disabling the button. Re-evaluate post-launch.
- **`?t=` deep-link is a 015 expansion, not a 020 internal hack.** It lives on the listen route as a generally-useful capability (other future features will want it too — e.g. "share this phrase moment" links). The Mis frases jump-to-source is the first consumer.
- **Frontend `normalizePhrase` mirrors backend, with a comment.** Drift is the failure mode; a comment + 1-line implementation on both sides keeps the contract honest until shared schema generation is worth it.
- **Hand-rolled Spanish relative-time formatter.** No `Intl.RelativeTimeFormat` dependency, full control over copy.
- **`gloss` field permanently dropped.** Per OVERVIEW §Non-Goals. 019's `ChunkResponse` already doesn't carry it; this just records that the Mis frases card design specifies a `chunk-gloss` slot which we deliberately omit.
- **`chunks` is a top-level route, not a child of `videos`.** Already true in `routes/`; reaffirmed.
